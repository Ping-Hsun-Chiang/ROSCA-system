# 標會（互助會）管理系統

一個以 **標會運作管理** 與 **繳款追蹤** 為核心的單頁應用程式，目標是將傳統紙本記帳的標會流程數位化，讓會首能輕鬆管理多個標會、自動計算每期應繳金額，並即時掌握每位成員的繳款狀態。

## Website
- Live Site: [https://ping-hsun-chiang.github.io/ROSCA-system/](https://ping-hsun-chiang.github.io/ROSCA-system/)

## Project Overview
本專案以純前端靜態網頁為基礎，採用 Vue 3 Composition API 搭配 Tailwind CSS，打造出無需後端即可完整運作的標會管理工具。資料透過 LocalStorage 持久化保存，適合會首在手機或電腦上隨時操作。

核心設計理念：
- 介面簡潔直覺，適合年長使用者操作
- 支援內標與外標兩種計息模式，自動精準計算
- 每期開標後即時彈出繳款確認介面，貼近實際收款流程
- 標會結束後自動產出各成員盈虧統計報表

## Main Features

### 1. 多標會管理
- 儀表板總覽所有標會進度
- 支援建立、刪除多個獨立標會
- 顯示標會名稱、模式、進度、繳款頻率等摘要資訊

### 2. 標會建立流程
- 設定標會名稱、計息模式（內標／外標）、基本會款
- 選擇開標日與繳款頻率
- 動態輸入會腳名單
- 建立前提供確認頁面，可返回修改

### 3. 開標與期數推進
- 首期由會首無息得標，系統自動處理
- 後續各期可指定得標者與標金，系統即時試算預覽
- 開標確認後彈出 Modal，可立即勾選繳款狀態與輸入各人備註
- 支援「完成」與「暫存」兩種操作模式

### 4. 繳款追蹤
- 每位成員旁附有 Checkbox 勾選是否已繳款
- 每位成員可獨立輸入備註
- 歷期紀錄預設為瀏覽模式，點擊「編輯」才可修改

### 5. 統計報表
- 標會結束後自動產出統計報表
- 顯示各成員得標期數、標金、總領取、總繳出與盈虧分析

## 計算公式

### 內標制（Inner Bidding）

> 核心概念：得標利息從活會的應繳金額中「扣除」。

| 身份 | 每期應繳金額 |
|------|-------------|
| 活會（尚未得標） | `基本會款 − 當期標金` |
| 死會（已得標） | `基本會款` |
| 得標者該期可領取 | `Σ 所有其他成員的應繳金額` |

**範例**（基本會款 10,000 元，當期標金 500 元，共 10 人）：
- 活會每人繳：10,000 − 500 = **9,500 元**
- 死會每人繳：**10,000 元**
- 假設目前有 3 位死會、6 位活會（得標者除外）：
  - 得標者可領取：(3 × 10,000) + (6 × 9,500) = **87,000 元**

### 外標制（Outer Bidding）

> 核心概念：得標利息加到死會的應繳金額上，且**各死會成員繳的利息為自己當初得標時的標金**。

| 身份 | 每期應繳金額 |
|------|-------------|
| 活會（尚未得標） | `基本會款` |
| 死會（已得標） | `基本會款 + 該成員當初得標時的標金` |
| 得標者該期可領取 | `Σ 所有其他成員的應繳金額` |

**範例**（基本會款 10,000 元，共 10 人）：
- 活會每人繳：**10,000 元**
- 死會 A（當初標金 800）繳：10,000 + 800 = **10,800 元**
- 死會 B（當初標金 600）繳：10,000 + 600 = **10,600 元**

### 首期（第一期）
- 會首無條件得標，標金為 0
- 所有會腳繳交基本會款
- 會首可領取：`基本會款 × 會腳人數`

### 最後一期
- 僅剩最後一位活會，標金可為 0
- 該成員自動得標
